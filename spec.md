# 仮想タッチパッドアプリ 仕様書

## 1. 概要

### 1.1 目的

Surface Proのようなタッチパネル搭載のWindowsクラムシェルPCを、狭い空間で使用する際に、物理タッチパッドの位置が手前すぎて操作しづらい問題を解決する。画面上にフローティングウィンドウとして仮想タッチパッドを表示し、タッチパネル経由でマウスカーソルを間接操作できるようにする。

### 1.2 ターゲットデバイス

- タッチパネル搭載のWindows PC（Surface Pro等）
- キーボード付きクラムシェルスタイルで利用
- 高DPIディスプレイ（200%スケーリング等）を想定

### 1.3 ユースケース

- 狭い机・膝上・車内などで物理タッチパッドに手が届きにくい状況
- タッチ操作では小さなボタンやメニューの精密な操作が困難な場面
- 外部ディスプレイ接続時のカーソル操作

### 1.4 Windows標準仮想タッチパッドとの差別化

Windows標準は「タブレット＋外部ディスプレイ」向けに設計されており、以下の制約がある。

- 半透明非対応（画面占有が大きい）
- サイズ変更不可
- 感度・加速度のカスタマイズが限定的

本アプリはクラムシェルスタイルでの常用を前提に、これらの制約を解消する。

### 1.5 推奨技術スタック

- **言語/フレームワーク**: C# + WPF
- **マウス制御**: Win32 `SendInput` API（P/Invoke）
- **タッチ入力**: WM_POINTER / WM_TOUCH メッセージ
- **常時最前面**: `HWND_TOPMOST`

## 2. UIレイアウト

### 2.1 ウィンドウ構成

```
┌─────────────────────────────────┐
│ ≡  ドラッグハンドル    [S][M][L] ─ × │  ← タイトルバー（ウィンドウ移動専用）
├─────────────────────────────────┤
│                                 │
│                                 │
│        タッチパッド領域           │  ← カーソル操作に反応する領域
│        （半透明）                │
│                                 │
│                                 │
├────────────────┬────────────────┤
│   左クリック    │   右クリック    │  ← 明示的クリックボタン
└────────────────┴────────────────┘
```

### 2.2 各領域の役割

| 領域 | 機能 |
|---|---|
| タイトルバー | ウィンドウのドラッグ移動。≡メニュー、サイズ切替ボタン（S/M/L）、最小化（─）、閉じる（×）を配置 |
| タッチパッド領域 | タッチ入力を受け取り、カーソルの相対移動・ジェスチャーを処理する主要領域 |
| 左クリックボタン | タップで左クリックを発行（ジェスチャーが苦手なユーザー向けのフォールバック） |
| 右クリックボタン | タップで右クリックを発行（同上） |

### 2.3 タイトルバーとパッド領域の分離

タイトルバー上のタッチはウィンドウ移動として処理し、パッド領域のタッチはカーソル操作として処理する。これにより「パッドを操作中に誤ってウィンドウが動く」問題を防止する。

## 3. ウィンドウ特性

### 3.1 常時最前面

- `Topmost = true`（WPF）/ `HWND_TOPMOST`（Win32）で他のウィンドウより常に前面に表示
- タッチキーボード等、他の常時最前面ウィンドウとのZ-order競合に注意

### 3.2 半透明

- パッド領域の不透明度を30%〜100%の範囲で調整可能
- デフォルト: 60%
- 下のUIを視認しつつ操作できる程度の透過

### 3.3 サイズ段階

| サイズ | 幅 × 高さ（論理ピクセル） | 用途 |
|---|---|---|
| S | 200 × 150 | 最小限の操作。画面占有を最小化 |
| M | 300 × 220 | 標準的な操作（デフォルト） |
| L | 400 × 300 | 広い操作面が必要な場合 |

- タイトルバーの [S][M][L] ボタンでワンタッチ切替
- 設定画面でカスタムサイズも指定可能

### 3.4 位置記憶

- ウィンドウ位置を終了時に永続化し、次回起動時に復元
- スリープ復帰後も位置を維持

### 3.5 タスクトレイ常駐

- タスクバーには表示しない
- システムトレイ（通知領域）にアイコンを常駐
- トレイアイコンの右クリックメニュー: 表示/非表示切替、設定、終了

## 4. 入力モデル

### 4.1 カーソル移動方式

**相対移動モデル（物理タッチパッド方式）**を採用する。

- 指を置いた位置ではなく、指を動かした量（デルタ）でカーソルを移動
- 加速度カーブを適用し、遅い動きは精密操作、速い動きは大きな移動に対応
- `SendInput` API の `MOUSEINPUT`（`MOUSEEVENTF_MOVE`）でカーソル移動を発行

### 4.2 ジェスチャー一覧

#### 基本操作（Phase 1-2 / 必須）

| ジェスチャー | 動作 | 実装フェーズ |
|---|---|---|
| 1本指ドラッグ | カーソルの相対移動 | Phase 1 |
| 1本指タップ（短押し） | 左クリック | Phase 1 |
| 1本指ダブルタップ | ダブルクリック | Phase 1 |
| 2本指タップ | 右クリック | Phase 2 |
| 2本指ドラッグ（上下） | 垂直スクロール | Phase 2 |
| 2本指ドラッグ（左右） | 水平スクロール | Phase 2 |
| タップ→ホールド→ドラッグ | ドラッグ＆ドロップ | Phase 2 |

#### 拡張操作（Phase 4）

| ジェスチャー | 動作 |
|---|---|
| 2本指ピンチ | ズーム |
| 3本指タップ | 中クリック（設定変更可） |
| 3本指スワイプ上 | タスクビュー表示 |
| 3本指スワイプ下 | デスクトップ表示 |
| 3本指スワイプ左右 | アプリ切替 |

### 4.3 ドラッグ＆ドロップのステートマシン

```
IDLE
  │ (指がパッド領域に接触)
  ▼
TRACKING
  ├── (短時間で離す、移動量小) → TAP（左クリック発行）→ IDLE
  ├── (即座に移動開始) → CURSOR_MOVE → (指離す) → IDLE
  └── (一定時間ホールド、移動なし) → HOLD_READY
                                        │ (指移動開始)
                                        ▼
                                      DRAGGING（mousedown発行済み）
                                        │ (指離す)
                                        ▼
                                      DROP（mouseup発行）→ IDLE
```

**パラメータ（設定可能）**:
- タップ判定の最大移動量: 10px（デフォルト）
- タップ判定の最大時間: 200ms（デフォルト）
- ホールド判定の時間閾値: 500ms（デフォルト）
- ダブルタップの間隔上限: 300ms（デフォルト）

### 4.4 タッチ→マウス変換の自己参照問題への対策

Windows はタッチ入力を自動的にマウスイベントに変換するため、仮想パッド領域のタッチで二重入力が発生する。

**対策**:
- `RegisterTouchWindow` でパッド領域のタッチを自前処理に登録
- WM_POINTER メッセージを処理し、`DefWindowProc` に渡さないことでOS側のタッチ→マウス変換を抑制
- `SendInput` で発行するマウスイベントのみがカーソルを動かす状態にする

## 5. 設定項目

### 5.1 カーソル動作

| 設定 | 内容 | デフォルト |
|---|---|---|
| ポインタ速度 | 1（低感度）〜 10（高感度）の10段階 | 5 |
| 加速度カーブ | リニア / Windows標準準拠 / カスタム | Windows標準準拠 |
| タップ感度 | タップ判定の閾値（時間・移動量） | 中 |
| スクロール方向 | 通常 / ナチュラル（反転） | 通常 |
| スクロール速度 | 1〜10 | 5 |

### 5.2 外観

| 設定 | 内容 | デフォルト |
|---|---|---|
| 不透明度 | 30%〜100%（スライダー） | 60% |
| テーマ | ダーク / ライト / システム準拠 | システム準拠 |
| パッドサイズ | S / M / L / カスタム数値 | M |

### 5.3 動作

| 設定 | 内容 | デフォルト |
|---|---|---|
| Windows起動時に自動スタート | スタートアップ登録 | OFF |
| 起動時に自動表示 | 起動直後にパッドを表示 | ON |
| 表示/非表示ホットキー | キーボードショートカットで切替 | Ctrl+Alt+T |

## 6. 技術的考慮事項

### 6.1 タッチとペン入力の区別

Surface ProはSurfaceペンに対応している。WM_POINTERの`pointerType`フィールドで入力種別を判定する。

- `PT_TOUCH`: タッチ入力 → パッド操作として処理
- `PT_PEN`: ペン入力 → 無視（パッド操作として処理しない）
- 設定でペン入力にも対応するオプションを検討

### 6.2 パッド外タッチとの非干渉

仮想タッチパッドのウィンドウ外でのタッチ操作は、Windowsの通常のタッチ動作（直接操作）として処理されなければならない。アプリはパッド領域内のタッチのみを横取りし、それ以外に一切干渉しない。

### 6.3 DPIスケーリング対応

- アプリをPer-Monitor DPI Awareとして宣言（appマニフェスト）
- サイズ指定・カーソル移動量の計算をすべて論理ピクセルベースで行う
- DPI変更イベント（`WM_DPICHANGED`）でウィンドウサイズとレイアウトを再計算

### 6.4 画面回転への対応

- `DisplaySettingsChanged` イベントを監視
- 回転後にウィンドウ位置がスクリーン外にはみ出す場合、自動的に画面内に補正

### 6.5 マルチモニター対応

- `SendInput` のカーソル座標は仮想デスクトップ全体の座標系で計算
- ウィンドウ位置の記憶はモニター構成の変化を考慮（モニター取り外し時の補正）

### 6.6 UAC / セキュリティ制約

- 管理者権限で動作するウィンドウ（UAC昇格済み）に対しては、一般権限からの`SendInput`がブロックされる
- 完全な互換性が必要な場合: アプリを管理者権限で起動 or UIAccessマニフェスト＋署名付きデプロイ
- Phase 1では一般権限で起動し、制約がある旨をドキュメントに記載

### 6.7 タッチキーボードとの共存

- 両方が同時表示された場合に重なり合わないよう配慮
- タッチキーボードの表示/非表示イベントを監視し、必要に応じてパッド位置を自動退避（将来検討）

### 6.8 スリープ復帰後の状態復元

- ウィンドウ位置・サイズ・表示状態をファイルまたはレジストリに永続化
- スリープ復帰時（`WM_POWERBROADCAST`）に状態を再適用
